# ADR 001: Выбор стратегии развертывания ML-модели

## Контекст
Необходимо развертывать новую версию ML-модели без простоя сервиса.

## Рассмотренные стратегии
- Rolling update
- Blue-Green
- Canary

## Решение
Выбрана стратегия Blue-Green.

## Обоснование
Blue-Green позволяет держать старую версию активной до полной проверки новой. При ошибках rollback занимает секунды.

## Риски
- Удвоение ресурсов
- Требуется балансировщик

## Последствия
Нужен Nginx или Traefik для переключения. Затраты на два окружения.
